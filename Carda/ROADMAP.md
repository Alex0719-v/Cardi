## P4：名片交换

1. 交换技术方案验证 `[done]`
- 目标：确认第三方 App 可实现的近距离交换方案。
- 当前方案：不调用 AirDrop、NameDrop 或 NFC；双方都打开 Carda 并停留在“我的名片”页面时，使用 Multipeer Connectivity 自动发现同款 App、建立连接并传输 Nearby Interaction discovery token 与名片数据；使用 Nearby Interaction 读取 distance，并以 5cm 作为交换距离门槛；这里的“碰一碰”指距离足够靠近，不指物理碰撞或加速度碰撞。
- 验收：工程已接入 `Services/CardExchange/`，并在 `Info.plist` 中配置本地网络权限与 Bonjour service `_carda-ex._tcp`。

2. 双方交换 `[doing]`
- 前置：双方都打开 Carda，且停留在“我的名片”页面并各自存在当前名片。
- 目标：两台 iPhone 在 Carda 页面内靠近到 5cm 时交换当前展示名片。
- 当前：首版功能已改为 5cm 距离靠近触发。页面进入可交换状态后自动启动 Multipeer advertiser/browser 与 Nearby Interaction session；连接后交换 Nearby Interaction discovery token 并启动 `NISession`；当最新 distance <= 0.05m 时发送当前名片 payload。发送当前名片时播放 App 内递出名片动画；递出动画完整结束后停顿 0.1 秒，再播放接收名片动画。接收态按 Figma `10050:2742` 叠加背景模糊、左右按钮和对方名片，对方名片目标位置为 x=16、y=326、w=370；3 秒未拒绝自动接收，并在收纳到左下角名片夹图标方向的动画结束后以 `ownerKind = .received` 存入名片夹；发送和接收分别按 peer 做 5 秒冷却，避免重复触发；灵动岛 / Live Activity 方案已暂停。
- 修复：Nearby Interaction timeout 后会重新 `run` 当前 session；session invalidated 后会重建 `NISession` 并重新发送 discovery token，避免远离后卡在“近距离测距已停止”且无法再次靠近触发。
- 验证：DEBUG + Simulator 下已加入 `CARDA_ENABLE_EXCHANGE_SIMULATION=1` / `CARDA_AUTO_SIMULATE_EXCHANGE=1` 模拟入口，可用假 4cm distance 触发收到名片写入链路；该入口只验证 payload 保存为 `received` 的链路，不代表真实 UWB 验证。
- 剩余：需要两台支持 Nearby Interaction 精确测距的 iPhone 做端到端真机验证；模拟器不能验证真实 UWB distance。
