# Cardi Design System

> 本文件记录 Cardi 的设计系统。当前依据用户描述与 Figma `交互名片 Copy` Page 1 建立。

---

## 1. 设计原则

- 以名片为视觉中心。
- 页面背景保持轻、灰、低干扰。
- 核心控件使用玻璃质感。
- 主色只用于强调当前状态、选中状态或关键交互。
- 名片本体需要清晰、克制、可保存为独立图片。
- 设计实现以 Figma frame 为准，不自行发明新的视觉语言。

## 4. 字体

当前 Figma 样式：

| 样式 | 字体 | Size | Line height | Weight | 用途 |
| --- | --- | --- | --- | --- | --- |
| H1 | PingFang SC | 34 | 41 | Bold / Semibold | 页面标题、编辑页姓名 |
| CardName | PingFang SC | 33 | 35 | Semibold | 名片姓名 |
| CaptionCN | PingFang SC | 15 | 20 | Regular | 表单字段标题 |
| BodyCN | PingFang SC | 17 | 22 | Regular | 表单中文内容 |
| CardBodyCN | PingFang SC | 14 | 20 | Regular / Medium | 名片职位、公司、地址 |

| ToolbarText | SF Pro | 17 | 22 | Semibold / Medium | 系统控件与导航 |

实现规则：
- 中文文本必须使用苹方 / PingFang SC。
- 英文、拼音、电话、邮箱、链接等拉丁字符信息必须使用 SF Pro。
- SwiftUI 实现中，中文入口优先使用 `CardaTheme.pingFang(...)`，英文入口优先使用 `CardaTheme.sfPro(...)`。
- 不新增自定义字体，除非用户确认。
- Figma 中部分 Apple 模板文本带有系统字距设置；实现时优先使用系统默认字距，不手动添加负字距，除非用户明确要求像素级覆盖。
- 新版完整名片只替换右侧按钮与卡片轮廓；左半部分沿用上一版样式：职位为 PingFang SC Regular 14pt，姓名为 PingFang SC Semibold 33pt、约 4.8pt 字距，英文名/拼音为 SF Pro Regular 14pt。姓名固定单行显示，内容超过 132 x 35pt 区域时沿用原有最小缩放；不再针对四字姓名单独调整字距或视觉字号。
- 除姓名外，名片上所有文字统一为 14pt Regular；中文用 PingFang SC，英文/拼音/电话/邮箱/链接用 SF Pro。

- 激活搜索时，搜索圆形展开成横向搜索栏，左侧导航胶囊收缩为单按钮。
- 底部导航 Liquid Glass 的轮廓与图层来源为 Figma `10063:3134`，左侧玻璃节点为 `10063:3136...3139`，右侧搜索玻璃组件为 `10063:3176`。Figma 原始三层 Fill 为 65% 白色 Normal、`#DDDDDD` Color Burn、`#F7F7F7` Darken；按最新增强指令，当前实际可见强度改为 44% / 48% / 42%，让原生玻璃折射与背景取色更明显。玻璃基底仍为 0.4% 黑色；阴影增强为 x=0、y=10、blur=44、spread=0、16% 黑色。
- Figma GLASS effect 原始参数为 radius=7、refraction=1、depth=16、lightAngle=315°、lightIntensity=0.8、dispersion=0、splay=0.06；cornerSmoothing=0.6。Figma 没有普通 stroke，边缘描边由 GLASS effect 的高光折射生成。SwiftUI 必须使用不带额外 tint 的原生 `.glassEffect(.regular, in: .rect(...))` 对应 Figma 的 iOS 26 `Liquid Glass / Regular / Small`，不得另加常规描边；可交互表面追加 `.interactive()`。
- iOS 26 及以上时，左侧导航、搜索按钮/搜索栏与清空按钮置于同一个 `GlassEffectContainer(spacing: 0)`。零静止融合距离仍允许容器批量渲染玻璃，但不会让相距 16–20.5pt 的搜索框、返回按钮或清空按钮在页面进入时自动吸引和形变；每个可点击表面继续使用 `.interactive()`，只在真实按压时显示系统点击反馈。其他公共玻璃按钮取消额外白色 tint，继续使用原生 `.regular`。Cardi 最低支持 iOS 17.0；低于 iOS 26 时不得引用 Liquid Glass API，必须降级为浅色 `ultraThinMaterial` / 毛玻璃，白色覆盖降为 22%，并继续用原有细线模拟玻璃边缘。
- 选中 tab 保留原本的灰色滑块，填充为不透明 `#EDEDED`，不绘制额外白色描边；滑块位于 Liquid Glass 导航表面之上。
- tab 页面切换反馈：新选中的 SVG 使用四阶段 `idle → compressed → expanded → settled`，0.08 秒压到 scale=0.90 / y=+0.8，随后 0.17 秒低 bounce 弹到 scale=1.045 / y=-0.4，再用 0.15 秒回到 scale=1 / y=0。整条 402 x 95 底栏同步从 scaleX=0.996 / scaleY=0.978 / y=+0.6 回弹到 scaleX=1.002 / scaleY=1.006 / y=-0.2，再归位。灰色滑块参考登录/注册页的原生 segmented control，单独使用 `.smooth(duration: 0.24, extraBounce: 0)` 平滑移动；减弱动态效果时立即切换，不使用回弹。
- 反馈只由 `selectedSection` 的真实变化触发；重复点击当前 tab、搜索展开/收起和键盘态移动不得误触发。开启“减弱动态效果”时不播放缩放和位移回弹。

## 6. 名片视觉系统

### 6.1 名片基准

来源：Figma `9978:2322`（新版带右侧按钮名片）

| 属性 | 值 |
| --- | --- |
| 位置 | x=16, y=157 |
| 尺寸 | 370 x 222 |
| 圆角 | 26；裁切槽顶部两个受宽度限制的倒角为 20.5；全部使用普通 `.circular` / 圆弧圆角，不使用 Corner smoothing 或 `.continuous` |
| 背景 | white + 当前名片保存的 `Group 42.png` / `color2.png` / `color3.png` 模板 |
| 阴影 | 当前版本完整名片本体与名片右侧圆形操作按钮均不绘制阴影 |
| 底图图片 | `Group 42.png` 为 370 x 300；`color2.png`、`color3.png` 为 370 x 370；均居中裁切 |
| 覆盖层 | 无，不再叠加半透明图层 |
| 右下按钮槽 | 基准相对坐标 x=329, y=103, w=41, h=119 |

名片横向宽度固定为 370pt。短边基础高度为 222pt，随信息量增长。
名片底图不再使用 SwiftUI 自绘椭圆或渐变。三个稳定模板 `color1`、`color2`、`color3` 分别读取项目 `Card photo/Group 42.png`、`Card photo/color2.png`、`Card photo/color3.png`，按新版带按钮槽的名片轮廓 `scaledToFill` 居中并上下裁切。绘制顺序为先按动态高度裁切 PNG，再切右下按钮槽，再对最终轮廓统一倒角；当前版本完整名片本体不绘制阴影。底图上方不得再叠加半透明柔光、灰色遮罩或其他覆盖层。右侧 34pt 圆形操作按钮使用同一个完整底图渲染结果，按按钮相对名片的 x/y 坐标裁切后成圆，不使用纯白填充；按钮图标独立覆盖，纹理坐标必须与名片主体严格一致。

创建/编辑页顶部预览按 `color1 → color2 → color3` 固定顺序分页并在首尾停止。分页步长为整张 402pt 画板宽（370pt 名片 + 32pt 间距），使用与“我的名片”一致的 0.18 秒分页曲线，让新底图和右侧按钮底色从画板最侧边滑入；预览中的文字、头像、公司 Logo 和所有按钮图标作为固定前景层，不参与水平位移。名片正下方显示 3 个直径 8pt、间距 12pt 的圆点，活动点为白色、非活动点为 `#CFCED3`，不绘制外层跑道圆。预览绑定 `BusinessCardDraft.backgroundTemplate`：创建默认 `color1`，编辑从持久化选择开始，确认后同步进入 SwiftData、账户 JSON 归档/备份和交换 payload；旧数据缺少字段时回退到 `color1`。完整名片、名片夹、搜索与 PNG 导出统一从 `CardRenderData.backgroundTemplate` 渲染，不能只在编辑预览中替换图片。
裁切槽左边界相对上一版向右移动 7pt。除槽顶靠近最上方按钮的两个倒角外，其余倒角在原本 24pt 基础上增加 2pt，当前为 26pt；槽顶两个倒角在槽宽不足时自适应到槽宽的 1/2（当前最大为 20.5pt）。如果多按钮场景因槽高不足产生倒角冲突，应优先略微上提槽顶，而不是压小其他倒角；当右侧只有 1 个按钮 / 1 条信息时，裁切槽不使用该上提保护，槽顶相对上一保护位置向下移动 5.5pt。

```text

  + max(0, visibleInfoFieldCount - 3) * 24
  + wrappedExtraLineCount * 20
```

锁定关系：

- 头像顶部和右侧锁定。

## 上滑交换交互

### 递卡手势

- 手势必须从当前完整名片范围内开始，不允许页面空白区的纵向滚动误触发递卡。
- 纵向递卡识别挂在当前名片轮播的稳定命中面，起点与位移统一换算到“我的名片”402pt 画板命名坐标系；整页祖先层只保留横向分页识别。不得把递卡与分页分别追加为祖先层并列手势，否则名片点击/长按组合可能在真机上吞掉递卡回调。
- 上移 20pt 后进入短时发现状态，同时复制当前完整名片 UI 并覆盖到原名片上；之后只有复制层跟随手指向上移动，原页面名片不移动、不缩放、不隐藏。上移 120pt 或预测位移达到提交阈值后确认递卡，未达到阈值时复制层使用轻量弹簧回到原位并无缝移除。
- 横向名片分页与纵向递卡按主轴互斥：只有纵向位移大于横向位移 1.1 倍时才识别递卡；一旦递卡手势被锁定，横向分页不得继续移动。
- 提交后若目标仍未明确，手势副本立即回到原位并移除，短时发现继续在逻辑层运行；禁止把副本冻结在手势终点或页面上方等待 10 秒超时。之后锁定目标时，从原卡位置创建递出动画；提交时已经锁定目标则继续沿用当前副本。
- 找到明确目标时触发一次很轻的 selection 触感；该反馈只表示“已对准”，不得表示“已保存”。

### 目标反馈

- searching：`正在寻找附近可接收的人`
- direction unclear：`轻轻将手机朝向对方`
- ambiguous：方向模式显示 `目标不明确，请将手机朝向对方`；distance-only 模式表示附近最近两人距离过近，不得自动选择。
- locked：方向模式显示 `已对准{name}`；distance-only 模式显示 `已确认最近的{name}`。
- single sending：`正在递给{name}` / `等待对方接住`
- mutual sending：`正在交换名片` / `等待双方保存`
- persisted ACK：`{name}已接住`
- no target：`附近没有明确的可接收对象`

### 递出动画

上划 20pt 后创建与当前名片完全重合且具有稳定 ID 的动画层。它从 0.96 倍开始实时跟随手势，随 20...148pt 上划进度连续增长到 1.08 倍；按预测速度加入最大 5° 的 x 轴轻微姿态。锁定目标时用 `interactiveSpring(response: 0.24, dampingFraction: 0.86)` 向递出方向再吸附 12pt，并保持至少 1.05 倍。松手时若目标已锁定，前 0.15 秒把当前状态连续收至约 0.85 倍、-3.5°，随后 0.5 秒通过 SwiftUI `distortionEffect` 调用 `Components/CardGenieShader.metal`，使整张名片沿短弧线向屏幕上方逐行连续弯曲并收束到约 15% 宽度；若目标尚未锁定，则副本先用 0.2 秒回弹并移除，待逻辑层锁定后从原卡位置重新发起同款递出。底层原名片始终保持原位和完整可见；视觉上不得出现切片层叠、接缝、冻结副本或两张名片。

### 接收与翻面回递

接收名片状态以 Figma `10050:2742` 为准：背景叠加全屏实时模糊和 `rgba(158,158,158,0.5)` 灰色遮罩，模糊在约 0.3 秒内完成；左右上角显示 Liquid Glass 按钮，左侧为 `拒绝`，右侧为 `分到列表`。来卡从屏幕顶部 x=201、y=-18、约 15% 宽度的收束态开始，将 `CardGenieShader.metal` 的形变进度从 1 反向驱动至 0，在约 0.85 秒内沿短弧线连续展开为 x=16、y=326、w=370 的完整名片；完成后同一个 `GenieCardTransitionView` 保持在 progress=0 并直接承载翻转手势，不做视图替换或交叉淡化。3 秒内未拒绝则自动接收。普通收纳时把同一形变进度从 0 正向驱动至 1：靠近 x=163.5、y=822 名片夹图标的一侧先被吸引，另一侧随后跟进；名片沿弯曲路径连续收窄、压缩并在最后阶段淡出，总时长约 1.2 秒。禁止用多份 `BusinessCardView` 横向切片模拟形变。

“分到列表”使用与添加/修改列表一致的居中玻璃弹窗语言：300pt 宽、36pt 圆角，标题为 `选择名片列表`，中部按列表排序显示 52pt 单选行，超过 4 行时在 208pt 高区域内滚动；列表区域不得叠加灰色内容块或独立材质，必须透明继承弹窗本身的玻璃材质与颜色。底部依次显示蓝色 `确认` 与红色 `取消` 胶囊按钮；未选择列表或没有现有列表时确认不可用，空状态显示 `暂无现有列表 / 请先在名片夹创建列表`，点击取消或弹窗外遮罩可关闭。弹窗出现时在既有接收遮罩上追加 18% 黑色遮罩，并暂停、重置 3 秒自动接收计时；确认后先关闭弹窗，再沿原奇幻收纳路径保存到所选列表。

只有单向 `delivery` 来卡显示 `向侧边翻转名片可回递`。用户在来卡上横向拖动达到 84pt 或翻转进度超过 126°后完成；视觉采用横向压缩到窄边、切换为本机当前名片、再展开的翻面方式，完成段约 0.5 秒。翻面后停顿约 0.12 秒并先保存来卡、发送 `persistedAck` 和新 `returnDelivery`；随后本机名片从接收位置以完整不透明的连续奇幻形变向上递出，释放阶段约 0.14 秒、形变阶段约 1.05 秒。翻面回递不再并行播放对方名片的收纳动画，保证递出是唯一主视觉；普通接收、自动接收与“分到列表”继续使用奇幻收纳。`mutual` 和 `returnDelivery` 来卡不显示翻面入口。

交换动画中的名片和名片右侧圆形按钮不绘制阴影。

### 持久化成功反馈

- 数据发出、网络收到或接收动画开始均不得触发“已接住”成功反馈。
- 只有收到对方 `persistedAck` 后显示“对方已接住”。
- 单向递卡成功使用一次 `UINotificationFeedbackGenerator(.success)`。
- 双向交换中本机收到对方对当前名片的持久化 ACK 时，使用两次间隔约 0.12 秒、强度克制且对称的 medium impact，表达该次互换的双向语义。
- 拒绝、目标歧义、连接中断或保存失败不得使用成功触感。

灵动岛 / Live Activity 方案当前暂停，工程不保留对应 Widget Extension、Activity attributes 或系统开关。后续若恢复该需求，应先重新确认系统能力边界和视觉方案。

标题/头像栏上方 x=0、y=0、w=402、h=62 的 `TransparentGradientBlur(direction: .top)` 过渡层已完全移除，该区域不得增加替代材质、纯色色块或安全区补偿层。列表模式收起态仍允许整个 Sticky Header 模块移动到 y=54，Header 悬顶 top 固定为 y61、名片裁切线为 y66；顶部栏与 AppShell 背景的既有 62pt 可见性裁切逻辑保持不变。

Cardi 当前设计系统只定义浅色外观。`ContentView` 必须在应用根层使用 `.preferredColorScheme(.light)`，确保 iOS 17+ SwiftUI Material、`systemChromeMaterial` 与 iOS 26+ Liquid Glass 在手机开启自动/手动深色模式时仍采用浅色材质配方。不得只固定页面背景和文字颜色而让系统材质继续跟随深色模式，否则渐变毛玻璃会变成深灰/黑色，其上方的玻璃控件也会被错误背景染灰。

姓名/公司模式滚动视口固定为 y=54...874；展开态使用 110pt 顶部占位，使内容 body 仍从 y=164 开始，分组标题起始在 x=16、y≈188，首张折叠名片约 y=220。顶部收起偏移 `headerCollapseOffset` 只取 0 或 110pt，标题、头像、分段切换栏、AppShell Union、分组裁切上沿与列表 Sticky Header 模块由同一偏移驱动。列表顶部 26pt 可滚同色区域继续保留初始布局，但不参与导航栏方向门槛。
列表/姓名/公司模式共用二态手势规则。方向唯一读取当前 `UIScrollView.panGestureRecognizer` 的逐帧 translation delta：手指上推、浏览页面更下方内容累计约 1pt 后直接收起到 110pt；手指下拉、浏览页面更上方内容累计约 1pt 后直接展开到 0。列表模式的向上收起判定额外要求 `expandedListID` 非空；全部列表收起时不得写入 110pt 顶部状态。状态转换统一使用 0.20 秒 `cubic-bezier(0.2, 0.72, 0.18, 1)`；方向反转可以中断尚未结束的动画，但同方向帧不重复发起动画。
iOS 17 兼容滚动桥仅记录 clamp 后的可见内容位置并处理程序化 offset 请求，不参与方向判定。原生 pan 结束后不会再产生方向事件，因此布局变化、程序化滚动、减速惯性与触底弹性回弹均不能反向改写顶部状态；禁止恢复 ScrollPhase、滚动/标题锚点投影、速度/进度端点吸附、时间 suppression、列表专用方向状态或标题动画期间的内容 offset 补写。进入名片夹、跨页面返回、退出搜索或切换列表/姓名/公司模式时，`headerCollapseOffset` 必须无动画重置为 0。
顶部导航文字透明度使用同一个可中断的 `headerCollapseOffset` 展示值计算：`progress = clamp(offset / 110, 0...1)`，`progress <= 0.5` 时 opacity=1，之后按 `(progress - 0.5) / 0.5` 线性淡出到 0。不得把计算后的 0/1 端点交给各文字节点分别做普通 opacity 动画，否则快速反向手势可能使文字透明度与标题位移脱节。标题、列表操作按钮和三个模式标签共用这一映射；0.20 秒曲线和既有视觉参数不变。
名片夹模式切换稳定性修复仅增加交互代际与分组锚点身份隔离；不修改任何 Figma 坐标、材质、层级或动画曲线/时长/延迟/透明度/位移。
AppShell Union、顶部导航与列表 Sticky Header 模块必须直接共享 `headerCollapseOffset` 的外部 `.offset` 变换；不得把该值作为 Union Shape 的独立 `animatableData`。列表模块在 12pt 内容裁切 mask 外绘制同色结构桥接层，保证快速滚动和端点动画期间 Union body 与 Header 之间没有透明断层。列表 LazyVStack/卡片内容不观察该高频值，观察范围限定在轻量几何修饰器。
右侧 A-Z/# 索引位于内容前景层，避免被任何覆盖层遮挡。
名片夹顶部列表/姓名/公司三个分段标签的文字位置必须与选中状态解耦，以姓名模式坐标为准固定中心：列表 x=65、姓名 x=201、公司 x=335，中心 y=145；切换模式时只允许背景 Union 轮廓变化，文字不得产生水平位移动画。
三个分段按钮的命中区域必须与当前模式下各自的 Figma 标签 frame 和 `HolderModeTabShape` 一致，覆盖完整白色标签表面，不得继续使用文字自身的 46 x 20pt frame。命中层透明且不参与绘制；文字中心、Union path、层级与现有动画参数保持不变。
右侧索引每个字母保留 11pt SF Pro Semibold 视觉，并使用更宽的透明点击区。点击字母时通过对应 Section ID 立即定位；按住索引上下滑动时，根据手指经过的字母连续定位，同一字母停留期间不重复触发。姓名模式按姓名拼音首字母定位，公司模式按公司名称拼音首字母直接定位同名字母 Section；无对应分组时保持当前滚动位置。公司模式 Section 标题为 A–Z/#，组内折叠名片按公司名称拼音升序、同拼音时按姓名拼音排序，空公司进入末尾 `#` 分组。分组列表底部透明留白至少覆盖一个可视滚动区减去 20pt 标题高度，使末尾分组也可对齐吸顶位置。

- 进入搜索页时，`morphingIdleControls` 必须持续存在到 0.68 秒主形变完成，再关闭动画交接到相同位置的 `searchActiveControls`；不得在 `isSearchActive` 变为 true 的第一帧直接替换视图。退出搜索页时先无动画交接回搜索态终点的 `morphingIdleControls`，再播放反向形变。交接使用 generation 防止快速重复点击的延迟任务覆盖新状态。
