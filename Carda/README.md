# Carda

Carda 是一个 iOS 社交类名片软件，主要用于名片创建、名片交换与名片管理。

> 协作规则：开发前优先阅读 `AGENTS.md` 与 `contributing_ai.md`。

---

## 产品概览

Carda 的核心对象是“名片”。用户可以创建自己的交互名片，通过 App 内 5cm 距离靠近式交互递交或交换名片，并在名片夹中管理收到的名片。

主要页面：
- 我的名片
- 名片夹
- 名片搜索

主要能力：
- 创建一张或多张自己的名片。
- 编辑姓名、英文名/拼音、职位、公司、头像、公司 logo 和联系信息。
- 根据输入内容动态生成名片布局。
- 将名片保存为 `.png` 图片。
- 在名片夹中按姓名、分类、公司查看收到的名片。
- 搜索名片，按姓名、单位和其他信息分类展示结果。
- 通过设备碰一碰递交或交换当前展示名片。

- 双方打开 App 并停留在“我的名片”页面时，已接入 App 内“5cm 碰一碰”名片交换功能：使用 Multipeer Connectivity 自动发现和连接同款 App，传输 Nearby Interaction discovery token 与名片 payload；使用 Nearby Interaction 读取对方 distance，并以 `distance <= 0.05m` 作为交换触发门槛。这里的“碰一碰”指两台 iPhone 在 Carda 页面内足够靠近，不指物理碰撞或加速度碰撞。触发后先播放自己的名片递出动画，完整结束后停顿 0.1 秒，再播放接收名片动画；接收态按 Figma `10050:2742` 显示全屏背景模糊与左右上角按钮，对方名片在 0.3 秒内用 S 型曲线滑到 x=16、y=326、w=370 的目标位置。若 3 秒内未点击“拒绝”，自动接收并在 0.3 秒内把名片用 S 型曲线缩小到 1/5，收纳到左下角名片夹图标方向，动画结束后以 `ownerKind = .received` 存入名片夹；交换动画中的名片和名片右侧圆形按钮不绘制阴影。
- 交换能力不调用 AirDrop、NameDrop 或 NFC；iOS 系统 NameDrop / Bringing Devices Together 不能被第三方 App 关闭，所以 Carda 不接管系统顶部贴合手势，而是在 App 内用 5cm 距离门槛确认交换。该能力只在“我的名片”页面有当前名片且未打开编辑页 / 添加弹窗时启用，离开页面会停止 Multipeer 和 Nearby Interaction session。同一 peer 的发送和接收各有 5 秒冷却，避免短时间重复触发。
- Nearby Interaction 测距被系统 timeout 移除时会自动重新 `run`；测距 session 意外失效时会重建 `NISession`、重新发送 discovery token，并恢复测距，不再把“近距离测距已停止”作为常驻提示展示。
- DEBUG + Simulator 中提供交换模拟验证入口：启动时设置 `CARDA_ENABLE_EXCHANGE_SIMULATION=1` 会显示“模拟碰一碰”按钮；同时设置 `CARDA_AUTO_SIMULATE_EXCHANGE=1` 会自动触发一次模拟交换，用于验证名片 payload 转换、`ownerKind = .received` 保存和 SwiftData 落库，不代表真实 UWB 测距验证。
- 工程已改为显式 `Info.plist`，并加入本地网络权限说明和 Bonjour service `_carda-ex._tcp`；灵动岛 / Live Activity 方案已暂停，当前工程不包含对应 Widget Extension。

尚未确认或待技术验证：
- App 内 5cm 碰一碰交换仍需要两台支持 Nearby Interaction 精确测距的真机做端到端验证；模拟器不能验证真实 UWB 距离。

## 当前项目结构

- `CardaApp.swift`：应用入口与 SwiftData 容器。
- `ContentView.swift`：进入 `AppShellView`。
- `Item.swift`：模板遗留模型，暂保留在 SwiftData schema 中以避免迁移风险。
- `Models/`：名片、字段、名片列表、草稿和字段类型。
- `Views/MyCards/`：我的名片、创建/编辑名片。
- `Views/CardHolder/`：名片夹。
- `Views/Search/`：名片搜索。
- `Components/`：名片渲染、折叠名片、底部导航、头像等复用组件。
- `Components/LocalSVGIconView.swift`：本地 SVG 图标缓存解析与 SwiftUI `Canvas` 渲染，用于底部导航和名片右侧操作按钮等高频图标。
- `Components/TransparentGradientBlur.swift`：统一封装名片夹与搜索页的满强度实时渐变毛玻璃模糊。
- `Services/`：保存名片图片、示例数据、默认列表补齐与近距离名片交换等服务。
- `Services/CardExchange/`：Multipeer Connectivity 发现/传输、Nearby Interaction token 测距、交换消息与名片 payload。
- `DesignSystem/`：颜色、尺寸、玻璃样式和名片常量。
- `My icon/`：底部导航 tab、名片右侧操作按钮与名片夹删除按钮的 SVG 图标源文件。
- `Card photo/`：名片底图 PNG 资源。
- `Assets.xcassets/`：资源目录。
- `Info.plist`：本地网络、Bonjour、相册保存与应用基础 plist 配置。
- `AGENTS.md`：项目级 AI 指导规则。
- `contributing_ai.md`：AI 协作与变更协议。
- `Designsystem.md`：设计系统记录。
- `ROADMAP.md`：路线图与待确认事项。
